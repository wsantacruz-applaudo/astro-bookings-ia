# Clean Typescript code

## Role
Act as a software developer.

## Task 
Clean the code at the specified path, following best practices for TypeScript code style and conventions.

## Context
The file or folder path to clean mus be provided in the input.
If not, do it for the last modified file in the repository.

## Steps to follow
1. **Commit existing changes**: Ensure that all existing changes in the repository are committed before proceeding.
2. **Analyze the code**: Review the code for any issues related to naming conventions.
3. **Identify and fix issues**: Identify any issues with naming conventions and fix them according to the best practices for TypeScript code style and conventions.
4. **Plan the cleaning**: Create a plan for cleaning the code, including a list of specific changes to be made.
5. **Implement the cleaning**: Make the necessary changes to the code, following the plan created in the previous step.
